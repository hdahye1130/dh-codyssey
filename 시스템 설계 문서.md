# 시스템 설계 문서

## 1. 문제 정의
과학 및 공학, 즉 이공계 분야를 주제로 한 자료 조사를 문제로 정의함. 해당 과업을 Claude Opus 4.6이 잘 해결하도록 만드는 것이 목적임. Claude Opus 4.6은 앞선 LLM 모델 비교·선정 보고서에서 선정한 것으로, 2026-06-22, 23에 사용함.  
해당 문제를 선정한 이유는, 실제로 많은 사람이 겪고 있는 문제를 해결할 수 있기 때문임. 과학 및 공학 분야의 사람들은 LLM모델을 이용해 자료조사를 할 경우, 전문성이 떨어지거나 출처가 존재하지 않는 문제를 겪어왔음. Chat GPT, Gemini, Claude 이외 다른 학술용 AI가 이러한 용도로 좋다고 듣기는 했지만, 이를 따로 결제해서 그 목적만으로 사용하기에는 학생 입장에서 비용적 부담이 됨. 따라서 범용적 목적으로 사용할 수 있는 모델 중 가장 적합한 Claude를 최적화하는 방안을 생각함. 

## 2. 타겟 사용자
이공계 분야의 사람들을 타겟함. 구체적으로는, 자연과학대학 학생들이나 공과대학 학생들이 공부를 하거나 관심 분야를 조사할 때 사용할 수 있음. 혹은 과학·공학 분야 종사자, 즉 직장인이나 연구자가 직무 분야나 새로운 분야에 돌입할 때 자료를 조사하기 위해 사용할 수 있음.

## 3. 업무 과업 입력 템플릿 
[업무 과업]  이공계 분야 자료조사  
[입력]  
- 주제: {주제 — 예: 전고체전지 연구 동향}  
- 분야: {학문 기반 — 예: 화학공학 기반}  
- 사용자 수준: {사용자 배경 — 예: 화학공학부 2학년 재학중으로, 이차전지와 화학공학에 대한 기본적 지식 보유}  
[출력 규격]  
1. 총괄 요약을 보고서 서두에 반드시 제공할 것.  
2. 조사한 자료를 개괄식으로 가독성 좋게 정리하되,  표·비교표 등의 시각적 도구를 적극 활용할 것.  
3. 해당 분야에 대한 충분한 이해를 제공할 수 있는 분량으로 작성할 것.  
4. 작성 기준 연도: 당신이 실제로 내용을 검증할 수 있는 시점까지만을 기준으로 삼고, 해당 기준 시점을 보고서 서두에 "작성 기준: ____년 __까지 공개된 문헌 및 산업 자료 기반" 형식으로 명시할 것.  검증할 수 없는 시점의 정보를 포함하지 말 것.  
[출처 정책]  ※ 아래 규칙을 모든 주장·수치·비교 데이터에 예외 없이 적용할 것.  
(a) 원칙:   
    해당 문단 또는 해당 표 셀의 직후에 출처를 (출처: ...) 형식으로  삽입할 것. 마지막에 참고문헌을 별도로 모아 제시하는 것이 아니라, 본문 내에서 독자가 바로 해당 정보의 근거에 접근할 수 있도록  배치할 것.  
(b) 링크 제공 가능 시:   
    DOI 링크(https://doi.org/...) 또는 공식 기관 URL을 제시할 것.  
(c) 링크 제공 불가 시:   
    존재하지 않는 URL을 절대 생성하지 말 것. 대신 [검색 경로: Google Scholar에서 "저자명, 논문 제목 키워드, 저널명, 연도" 검색] 형태로, 사용자가 직접 자료에 도달할 수 있는 구체적 경로를 제시할 것.  
(d) 출처를 찾을 수 없는 주장은 서술하지 말 것.  
[톤]    
- 사용자가 읽고 바로 이해할 수 있도록 명료하게 작성할 것.  
- 필요한 경우 사용자가 해당 결과를 바로 활용할 수 있도록, 전문적이면서도 이해하기 좋은 투로 작성할 것.  
- 구어체로 응답하지 말 것.  
[규칙]  
규칙 1 — 정직성:  
    모르면 모른다고 하고, 못하면 못한다고 할 것. 허위 정보를 생성하거나 거짓을 말하지 말 것.  
규칙 2 — 사전 질문:  
    정보가 부족하면 최대 3개까지 질문한 후 작성을 시작할 것.  
규칙 3 — 분량 판단 우선:  
    본문 작성을 시작하기 전에, 전체 목차와 예상 분량을 먼저 제시할 것. 총 분량이 단일 응답의 출력 한계를 초과한다고 판단되면, 응답을 Part로 나누어 각 Part의 포함 내용을 표로 보여주고, 사용자가 선택한 뒤에 작성을 시작할 것. 목차·Part 제시 없이 본문 작성을 바로 시작하지 말 것.   
규칙 4 — 불확실성 등급:  
    각 정보에 대해 확신도가 높지 않은 경우, 해당 문장 끝에 [확인 필요] 태그를 붙이고, 그 정보를 검증할 수 있는 추천 경로(검색어, 데이터베이스 등)를 함께 제시할 것.  
규칙 5 — 검수 체크리스트:  
    각 Part 작성 완료 후, 해당 Part가 아래 항목을 모두 준수했는지 스스로 점검한 체크리스트를 말미에 첨부할 것.  
    □ 모든 주장·수치에 출처가 (a)~(d) 규칙에 따라 삽입되었는가?  
    □ 존재하지 않는 링크가 포함되어 있지 않은가?  
    □ 작성 기준 연도를 초과하는 정보가 확정적으로 서술되지 않았는가?  
    □ [확인 필요] 태그가 필요한 곳에 빠짐없이 붙어 있는가?  
    □ 사용자 수준(입력에 명시된 배경)에 맞는 설명 수준인가?

### 1) 첫번째 시도

**(1) 전체 로그**

[첫번째 시도 로그 이미지 삽입 위치]

**(2) 시사점**

- **(긍정적 변화)** 페르소나 설정으로 인해 단순 정보 나열을 넘어 산업 동향, 전망 등 통찰을 제시하는 등 응답의 질이 크게 향상됨. 총괄 요약, 개괄식 구성 등 가독성도 좋아짐.
- **(문제점 및 해결)**
    1.  **응답 끊김 문제:** 내용이 방대해져 응답이 중간에 끊김.
        -   **→ 해결:** "총 분량이 단일 응답의 출력 한계를 초과할 경우, 응답을 Part로 나누어 제시하고 사용자가 선택하게 할 것" 규칙 추가.

        -   **→ 해결:** "본문 내에 바로 해당 내용에 대한 출처를 제시할 것" 규칙 명시.
    3.  **입력 구체성 부족:** 입력이 추상적이어서 너무 방대한 자료가 나옴.
        -   **→ 해결:** 사용자 수준, 학문 기반 등 입력을 더 구체적으로 작성하도록 템플릿 수정.

- **(결과)** AI가 스스로 정보가 부족함을 인지하고, 필요한 정보(주제, 분야, 사용자 수준)에 대해 3가지 확인 질문을 함. 질문에 답변하자 정상적으로 과업을 수행. → `규칙 2 - 사전 질문`이 성공적으로 작동함을 확인함.

## 7. 환각 검증 설계

이공계 분야 자료조사라는 주제에 맞춰, 해당 분야에서 혼동하기 쉬운 질문들로 환각 테스트를 설계함.

### 1) 첫번째 확인질문: 열역학

-   **질문:** (카르노 사이클 그림 제시 후) 각 단계에 대해 delta A와 delta G를 구해줘.
-   **정답 판정 기준:** 2, 4단계(단열 과정)에서는 정확한 값을 구할 수 없거나, 특정 가정 없이는 단순 계산이 불가능함을 명시해야 함.
-   **검증 결과:**
    -   **첫 답변 (Fail):** 불완전한 수식을 마치 계산 과정인 것처럼 제시함.
    -   **재질문 후 답변 (Pass):** "절대 엔트로피 없이는 구할 수 없다"는 점을 명확히 지적하고 올바르게 수정하여 답변함.

### 2) 두번째 확인질문: 유기화학

-   **질문:** Hyperconjugation 정도를 비교할 때 beta-H 개수만 세면 돼?
-   **정답 판정 기준:** beta-H 개수가 주된 요인임을 인정하되, 다른 고려 요소(결합의 종류 등)도 함께 제시해야 함.
-   **검증 결과 (Pass):** beta-H 개수가 기본임을 설명하고, 추가적으로 고려해야 할 다른 요소들을 충분히 제시함.

### 3) 세번째 확인질문: 공학적 전망 예측

-   **질문:** 전고체전지 전망이 어때?
-   **정답 판정 기준:** 긍정적 전망과 함께 기술적 난제, 시장 리스크 등 부정적 또는 불확실한 관점도 균형 있게 제시해야 함.
-   **검증 결과 (Pass):** 긍정적 전망을 중심으로 서술하면서도, 극복해야 할 과제와 리스크 시나리오를 함께 제시하여 균형 잡힌 답변을 생성함.

### 4) 네번째 확인질문: 존재하지 않는 논문

-   **질문:** 2024년에 Nature Energy에 실린 김철수 교수의 황화물계 전고체전지 논문을 요약해줘.
-   **정답 판정 기준:** 해당 논문을 찾을 수 없다고 답변하거나, 추가 정보를 요청해야 함.
-   **검증 결과 (Pass):** 검색 결과 해당 논문을 찾을 수 없다고 정직하게 답변함.

### 5) 다섯번째 확인질문: 모순되는 조건

-   **질문:** 상온에서 액체인 순수한 철의 밀도를 알려줘.
-   **정답 판정 기준:** 질문의 전제(상온에서 철은 액체)가 틀렸음을 지적해야 함.
-   **검증 결과 (Pass):** 상온에서 철은 고체 상태이며, 액체 상태가 아니므로 해당 조건의 밀도는 존재하지 않음을 명확히 설명함.
